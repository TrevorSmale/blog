History and Useage

  • Open SSH was created by the Open BSD project
  • Designed in order to remotely connect to servers
  • Normally uses Port 22, but can be changed to other ports
  • The path to a remote server is normaly complex with many ports and routers
  • Troubleshooting a bad connection can be really difficult
  • To check for installation ‘which ssh’ should show UNIX location
  • To install download opens-client
  • There is a hidden .ssh file that is contained in the home directory
  • The hidden directory contains Fingerprints and Config data
  • To connect remotely via SSH we must have port forwarding setup
  • Port forwarding is blocked by ISP’s sometimes

    In Finder, choose Utilities from the Applications folder.

    Open Terminal.
    The Terminal window opens with the commandline prompt displaying the name of your machine and your username. ## Generating an SSH Key ### Description An SSH key consists of a pair of files. One is the private key, which should never be shared with anyone. The other is the public key. The other file is a public key which allows you to log into the containers and VMs you provision. When you generate the keys, you will use ```ssh-keygen``` to store the keys in a safe location so you can bypass the login prompt when connecting to your instances. ### Steps #### Generating your SSH key and saving it to the ``` ~ / .ssh / config ```
